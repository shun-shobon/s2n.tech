## はじめに

Visual Studio Code（VS Code）は、拡張機能によってその真価を発揮します。
適切な拡張機能を選ぶことで、開発効率を劇的に向上させることができます。
今回は、開発者にとって本当に役立つ拡張機能を厳選し、用途別に詳しく紹介していきます。

## 必須の基本拡張機能

### 1. Prettier - Code formatter

**コードの自動フォーマット**

```json
// settings.json
{
  "editor.defaultFormatter": "esbenp.prettier-vscode",
  "editor.formatOnSave": true,
  "editor.formatOnPaste": true,
  "[javascript]": {
    "editor.defaultFormatter": "esbenp.prettier-vscode"
  },
  "[typescript]": {
    "editor.defaultFormatter": "esbenp.prettier-vscode"
  }
}
```

```javascript
// .prettierrc
{
  "semi": true,
  "trailingComma": "es5",
  "singleQuote": true,
  "printWidth": 80,
  "tabWidth": 2
}
```

**JavaScriptの静的解析とエラー検出**

## 生産性向上系拡張機能

### 1. GitHub Copilot

**AI駆動のコード補完**

```javascript
// コメントを書くとコードが提案される

function fibonacci(n) {
  if (n <= 1) return n;
  return fibonacci(n - 1) + fibonacci(n - 2);
}

// APIからユーザー情報を取得する関数
async function fetchUserData(userId) {
  try {
    const response = await fetch(`/api/users/${userId}`);
    const userData = await response.json();
    return userData;
  } catch (error) {
    console.error('Error fetching user data:', error);
    return null;
  }
}
```

## カスタムスニペット

### JavaScript/React スニペット

```json
// javascript.json (スニペット)
{
  "React Functional Component": {
    "prefix": "rfc",
    "body": [
      "import React from 'react';",
      "",
      "interface ${1:ComponentName}Props {",
      "  $2",
      "}",
      "",
      "const ${1:ComponentName}: React.FC<${1:ComponentName}Props> = ({ $3 }) => {",
      "  return (",
      "    <div>",
      "      $4",
      "    </div>",
      "  );",
      "};",
      "",
      "export default ${1:ComponentName};"
    ],
    "description": "React Functional Component with TypeScript"
  }
}
```

### API関数スニペット

```json
{
  "Async API Function": {
    "prefix": "apifunc",
    "body": [
      "export const ${1:functionName} = async (${2:params}) => {",
      "  try {",
      "    const response = await fetch(`${3:url}`, {",
      "      method: '${4:GET}',",
      "      headers: {",
      "        'Content-Type': 'application/json',",
      "      },",
      "      ${5:body: JSON.stringify(data),}",
      "    });",
      "",
      "    if (!response.ok) {",
      "      throw new Error(`HTTP error! status: ${response.status}`);",
      "    }",
      "",
      "    const result = await response.json();",
      "    return result;",
      "  } catch (error) {",
      "    console.error('API Error:', error);",
      "    throw error;",
      "  }",
      "};"
    ],
    "description": "Create async API function with error handling"
  }
}
```

## パフォーマンス最適化

### 拡張機能の無効化

```json
// settings.json
{
  // 特定の言語でのみ有効化
  "[python]": {
    "editor.defaultFormatter": "ms-python.black-formatter"
  },
  "[javascript]": {
    "editor.defaultFormatter": "esbenp.prettier-vscode"
  },
  
  // 不要な機能を無効化
  "git.enabled": false,  // Gitを使わないプロジェクトで
  "typescript.suggest.autoImports": false  // 重い場合
}
```

### ワークスペース固有の設定

```json
// .vscode/settings.json
{
  "search.exclude": {
    "**/node_modules": true,
    "**/dist": true,
    "**/.git": true
  },
  "files.watcherExclude": {
    "**/node_modules/**": true
  }
}
```

## まとめ

VS Codeの拡張機能を効果的に活用することで：

1. **コード品質の向上**: フォーマッタとリンタによる一貫性
2. **開発効率の向上**: スニペットと自動補完
3. **デバッグの効率化**: 統合されたツール群
4. **チーム開発の改善**: 共通設定と推奨拡張機能

拡張機能は適度に厳選し、プロジェクトに応じて使い分けることが重要です。
まずは基本的な拡張機能から始めて、必要に応じて追加していきましょう。
